# Contributing

Gracias por contribuir a UGasolineras.

## Flujo recomendado
1. Crea una rama desde `main`.
2. Haz cambios peque?os y enfocados.
3. Verifica compilacion debug local.
4. Abre Pull Request con descripcion clara.

## Convenciones
- Mantener compatibilidad Android API 26+.
- Evitar hardcode de secretos.
- Documentar cambios de comportamiento en README o docs.

## Validaciones minimas
- La app arranca sin crash.
- Geolocalizacion y carga de mapa funcionales.
- Cambio fosil/electrico funcional.
